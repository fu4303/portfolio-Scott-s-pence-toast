I made a starter blog with SvelteKit and GraphCMS, I've been doing a
couple of small projects with it and a _really_ like it!

Here's a small blog starter made with it from the GraphCMS blog
starter.

Create a directory for the project:

```bash
mkdir sveltekit-starter-blog
cd sveltekit-starter-blog
# initialise SvelteKit
npm init svelte@next
```

I chose the skeleton project option with no TypeScript support and
ESLint and Prettier config. Now I can install the dependencies with:

```bash
npm i
```

Now I can start the dev server and take a look at the project, I'll
start the dev server with `npm run` and add som additional paramerters
to the script with `--` then `-o` (or `--open`) to open in the browser
and `-p 3300` (or `--port 3300`) to specify the port to run on:

```bash
npm run dev -- -o -p 3300
```

Added Tailwind JIT:

```bash
npx svelte-add tailwindcss  --jit
```

Added a `__layout.svelte` file in the `routes` directory:

```html
<script>
  import '../app.postcss'
</script>

<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
</nav>

<slot />
```

Amended the `index.svelte` file to fetch posts from GraphCMS endpoint:

```html
<script context="module">
  import { GraphQLClient } from 'graphql-request'

  export async function load() {
    const graphcms = new GraphQLClient(
      import.meta.env.VITE_GRAPHCMS_URL,
      {
        headers: {},
      }
    )

    const { posts } = await graphcms.request(
      `{
          posts {
            slug
            title
          }
      }`
    )

    return {
      props: {
        posts,
      },
    }
  }
</script>

<script>
  export let posts
</script>

<h1>GraphCMS blog-starter</h1>
<ul>
  {#each posts as post}
  <li>
    <a class="text-blue-600 underline" href="/blog/{post.slug}">
      {post.title}
    </a>
  </li>
  {/each}
</ul>

<style style lang="postcss">
  /* your styles go here */
</style>
```

I'm still early days with Svelte but so far I'm super impressed with
it.
